## 📊 当前状态

4. **加密机制推断** - 生成AES密钥→AES加密数据→RSA加密密钥→发送

### 🎯 核心发现

#### collectData请求结构
```json
{
  "type": 1,
  "challenge": "s_xxx",
  "verifyId": "v_xxx",
  "collectData": "Base64(AES加密数据, 3000-4800字符)",
  "key": "Base64(RSA加密的AES密钥, 344字符)",
  "callback": "verify_xxx"
}
```

#### 加密流程
```
1. 前端生成随机AES密钥 (16字节hex)
2. AES-CBC加密行为数据 (collectData字段)
3. RSA-2048加密AES密钥 (key字段)
4. POST到 https://captcha.tuyacn.com/verify/v1/collectData
```

## 🚀 实施方案

### 方案A: 逆向完整加密逻辑 (高难度)
**优点**: 完全自主可控  
**缺点**: 需要深入逆向涂鸦验证码的加密实现  
**工作量**: 2-3天

**步骤**:
1. 在浏览器中精确定位AES加密函数
2. 提取加密参数(mode/iv/padding)
3. 提取RSA公钥获取方式
4. Node.js实现完整加密
5. Python集成测试

### 方案B: 浏览器自动化 + 云码打码 (推荐) ⭐
**优点**: 快速实现，成功率高  
**缺点**: 依赖浏览器环境  
**工作量**: 1天

**步骤**:
1. 使用Puppeteer/Playwright打开登录页面
2. 自动触发验证码
3. 截图验证码图片
4. 调用云码API识别缺口位置
5. 模拟滑动操作
6. 浏览器自动完成加密并提交
7. Python接管后续登录流程

### 方案C: 混合方案 (最佳)
结合方案A和B的优势：
1. 浏览器自动化完成验证码破解
2. 捕获加密后的collectData和key
3. Python直接使用这些数据进行验证
4. 后续登录完全自动化

## 🛠 技术栈

### 浏览器自动化
- **Puppeteer** 或 **Playwright**
- 无头模式运行
- 截图和模拟操作

### 验证码识别
- **云码API** (yunmaapi.com)
- 滑块验证码识别准确率>95%
- 响应时间<3秒

### 后端逻辑
- **Python** - 主控制流程
- **Node.js** - 可选的加密实现
- **Requests** - HTTP请求
